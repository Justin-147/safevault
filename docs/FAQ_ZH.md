# SafeVault 常见问题

## SafeVault 最久能恢复多久以前的文件？

只要某个版本仍被 SQLite 元数据引用，并且对应内容仍在 BLAKE3 对象库中，就可以
恢复。v1.1.5 的智能保留仍只提供规划和 dry-run，不会自动删除版本历史。因此实际
可恢复时间取决于开始保护的时间、是否成功捕获变化、人工维护操作和可用磁盘空间。

## 从未保护过的文件能恢复吗？

不能。SafeVault 不是裸盘恢复工具，无法恢复从未被快照或 watcher 自动保存捕获的
内容。

## SafeVault 会随 Windows 自动启动吗？

Windows 安装器默认启用当前用户的 daemon 和托盘开机启动项，安装时可以取消，
之后也可以移除。托盘中的“退出 SafeVault”会停止当前会话的后台保护；开机启动
设置决定下次登录时是否重新启动。

## 哪些文件夹会受到保护？

只有首次向导中选中的目录和后来手动添加的目录会受到保护。SafeVault 会在目录
存在时推荐 Desktop、Documents、Pictures；较大的项目目录会列出但默认不勾选，
也不会默认保护整个磁盘。可在首次向导或“保护目录”页面新增和取消目录。

## “停止自动保护”和“彻底移除历史记录”有什么区别？

“停止自动保护”只停止后续监听，已有快照、版本和恢复点仍然保留，重新添加相同路径
即可继续保护。“彻底移除历史记录”会删除数据库中的文件、版本、恢复点和事件索引，
之后无法再通过 SafeVault 恢复；它需要预览并输入目录 ID 或完整路径确认。

## 首次设置后可以关闭浏览器吗？

可以。浏览器只是本地恢复界面，daemon 会在后台继续初始扫描和监听。托盘中的
“退出 SafeVault”才会停止当前会话的后台保护、恢复首页和托盘。

## 为什么首次扫描占用了很多空间？

如果选择了包含许多项目、视频、数据集或生成文件的顶层目录，首个完整版本可能很
大。建议只保护具体的个人目录和项目目录，不要选择整个磁盘或大型工作区。取消保护
不会自动删除历史对象；在确认不再需要相关历史前，不要手工删除 `.safevault`。

## 可以把 SafeVault 数据从 C 盘移到其他磁盘吗？

可以。升级到 v1.1.5 后打开 Recovery Home 的“存储”页面，选择一个空目录，例如
`D:\SafeVaultData`。迁移会先复制数据库和对象并逐个校验，再切换位置。只有勾选删除
旧副本并输入 `MOVE STORAGE` 后才会释放旧位置；不要自己剪切 `.safevault`。

## 为什么设置 10GB 后占用仍可能超过 10GB？

10GB 是安全的软预算，不是硬上限。SafeVault 不会为了达标删除某个文件最后一份可
恢复内容。如果 Documents、Desktop 或项目目录中的最新文件合计已经超过 10GB，
就必须减少保护范围或移除可重新获得的大文件。对象库按完整文件内容寻址去重，并非
对文件内部块做差分；视频、安装包、模型和压缩文件通常也几乎无法再压缩。

## 如何恢复误删文件？

打开恢复首页，在“最近删除的文件”中找到目标并点击“恢复”。如果原位置已有文件，
SafeVault 会先保存当前内容。也可以进入历史页面选择时间点或恢复为新文件。

## 为什么刚删除的文件没有立刻显示？

恢复首页每 5 秒自动更新“最近删除”和“最近修改”。若几秒后仍未出现，请检查该目录
是否显示为“正在监听”。SafeVault 只能恢复删除前已经捕获的内容；刚创建后立即删除、
尚未来得及自动保存的文件不会有可恢复版本。

## SafeVault 是恶意软件沙箱或完整备份系统吗？

不是。命令沙箱主要防止项目目录被意外修改，不隔离凭据、网络或用户账户中的其他
文件。若要防止磁盘损坏、设备丢失或本地 vault 损毁，仍需保留离机备份。

## 异常批量修改保护能做什么？

SafeVault 会为高频批量变化和疑似加密扩展名记录重要恢复点与警告，且旧对象不会被
新内容覆盖。但它不承诺识别或阻止所有勒索软件行为。

## SafeVault 和 Git 有什么区别？

Git 主要保护已经提交的项目历史。SafeVault 保护选定目录中的本地文件版本，包括
未提交文件和非代码文档。代码项目可以同时使用两者。

## SafeVault 能替代 Time Machine、文件历史记录或云备份吗？

不能。SafeVault 聚焦选定目录的连续版本和误操作恢复，不提供整机或卷级保护，也不
提供云同步。建议继续使用系统备份，并把 SafeVault 导出文件保存到其他设备。

## 为什么应用 AI 沙箱结果时默认不删除文件？

删除属于高风险操作。`safevault apply` 默认只应用新增和修改；只有审查差异后显式
传入 `--allow-delete`，或在 GUI 中输入 `ALLOW DELETE`，才会应用删除。

## 为什么备份不能放在 SAFEVAULT_HOME 里？

如果备份和对象库位于同一位置，磁盘故障或误删可能同时破坏两者。备份目录也不能
放在受保护目录内，以免形成循环保护和不断增长的数据。

## GUI 能通过局域网或公网远程访问吗？

默认不能。GUI 只绑定 `127.0.0.1` 并使用随机 token，它是本地恢复界面，不是远程
管理控制台。不要把它暴露到公网。

## 为什么有些操作需要输入英文确认词？

普通恢复只需要本地确认框。删除、覆盖、导入和清理等高风险操作仍要求输入
`ALLOW DELETE`、`PRUNE`、`IMPORT` 等确认词，用来降低误点造成不可逆操作的风险。

## SafeVault 会不会一直占用更多空间？

相同内容只保存一次，但新版本仍会增加对象库用量。恢复首页会显示对象库用量和空间
预算。v1.1.5 的智能保留只提供规划和 dry-run，不会自动删除历史；应定期检查空间，
并在确认后清理未引用对象。
