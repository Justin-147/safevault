# SafeVault v0.2.0-rc1

SafeVault v0.2.0-rc1 is a release candidate, not a stable/final release.

## Highlights

- Automatic protection mode with `safevault protect`, daemon startup scans,
  dynamic watcher policy refresh, pause/resume, and recent deletion tracking.
- Recovery Home in the local GUI for recent deleted files, recent changes,
  search, one-click restore with local confirmation, and quick maintenance
  actions.
- Optional tray integration that opens the local GUI with the active session
  token instead of a bare URL.
- Automatic export backups with configured local time, target validation, and
  retry throttling after failures.
- Stronger root safety consistency across CLI, GUI, onboarding, snapshots, and
  daemon startup.

## Safety Notes

- SafeVault does not perform raw disk recovery and can restore only content
  previously captured by snapshots.
- Automatic protection never deletes user files.
- Backup targets are rejected inside `SAFEVAULT_HOME` and protected roots.
- Destructive apply, import, export overwrite, prune, unprotect, and sandbox
  cleanup flows still require explicit confirmation or dry-run handling.
- `safevault run` is not a hardened malware sandbox.

## Known Limitations

- The watcher is best-effort; snapshots remain the source of recovery truth.
- Export backups should be stored off-machine.
- Retention is planning-only in this release candidate.
- GUI is local-only and must not be exposed as a remote admin console.

## Release Gate

Before tagging this RC, confirm GitHub Actions are green on Linux, macOS, and
Windows, then run:

```bash
ruff check .
mypy src
pytest -q
python -m safevault --help
python -m safevault --version
bash scripts/release_check.sh
```
