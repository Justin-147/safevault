# SafeVault v1.0.1

SafeVault v1.0.1 fixes the Windows first-run problems reported after the 1.0.0
release. It does not change the BLAKE3 object store, SQLite metadata model, or
restore safety rules.

## What Changed

- Windows installer, Startup, tray, and Recovery Home launches no longer leave
  visible terminal windows.
- Packaged tray commands correctly launch packaged daemon and UI processes.
- First-run setup returns immediately; initial recovery points are built by the
  daemon in the background.
- Multiple custom protected folders can be added during onboarding, and large
  development workspaces are optional instead of preselected.
- Daemon status detects a missing process instead of displaying stale `running`.
- Tray Quit stops the daemon, local UI, and tray for the current session.
- Generated test/cache/review directories are ignored by default.
- The README status-badge 404 and browser favicon 404 are removed.

## Upgrade Notes

Install `SafeVaultSetup.exe` from this release over 1.0.0. Existing vault data,
protected-root metadata, and backups remain in place.

If 1.0.0 protected a very large workspace, review the folder list in Recovery
Home. Unprotecting a folder stops future monitoring but intentionally retains
existing history. SafeVault 1.0.1 does not automatically delete that history.
