# SafeVault v1.0.3

SafeVault v1.0.3 is a Windows monitoring reliability and Recovery Home cleanup
release. It fixes issues found during real first-run testing with several large
protected folders.

## What Changed

- Windows now detects the live background process correctly. Recovery Home no
  longer reports an active daemon as stopped because `os.kill(pid, 0)` is not a
  valid Windows process-liveness check.
- Folder watching starts before the startup reconciliation scan. Deletions and
  modifications are therefore observed while other large protected folders are
  still being scanned.
- Recent Deleted now contains one row for each file that is currently deleted.
  Restored files leave this list instead of continuing to occupy it.
- Recent modifications in Recovery Home are grouped by file and show the latest
  event, preventing one frequently written file from filling the panel.

## Recovery After Upgrade

Restart SafeVault after installing this version. Startup reconciliation checks
all tracked folders again. A file deleted during the earlier startup blind
window will appear in Recent Deleted if SafeVault captured a version before the
deletion. Files that were never captured cannot be reconstructed.

## Upgrade Notes

Install `SafeVaultSetup.exe` over 1.0.0, 1.0.1, or 1.0.2. Existing vault data,
protected-folder settings, and backup configuration remain in place.
