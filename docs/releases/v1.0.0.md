# SafeVault v1.0.0

SafeVault v1.0.0 is the first stable release of the continuous local file
protection workflow. It keeps the existing BLAKE3 object store, SQLite metadata,
snapshot/restore engine, and conservative sandbox apply boundaries.

## Highlights

- Windows installer with current-user daemon/tray Startup tasks and first-run
  onboarding.
- Continuous watcher-triggered versions, deletion markers, recovery timeline,
  search, and one-click restore.
- Recovery Center labels that hide snapshot IDs, version IDs, and object hashes
  during normal use.
- Before/after restore points for supported AI coding tools, including Codex and
  Cursor.
- Important recovery points and notifications for large or suspicious
  mass-change activity.
- Verified export/import, object-store integrity checks, and conservative
  dry-run/confirmation behavior.
- English and Chinese install, user, FAQ, recovery, and safety documentation.

## Install

Windows users should run `SafeVaultSetup.exe` and complete onboarding. The
installer enables daemon and tray Startup tasks by default; either choice can be
disabled during setup. Source installations require Python 3.12 and can use:

```powershell
python -m pip install -e ".[ui,tray]"
safevault ui --open
```

## Known Limits

- SafeVault restores only content captured after protection was enabled. It is
  not raw-disk recovery.
- The local command sandbox is not malware containment.
- Smart retention is planning/dry-run only in v1.0.0 and never deletes version
  history automatically.
- Mass-change detection is heuristic and does not detect every ransomware
  technique.
- Local objects do not protect against disk failure; keep verified exports or
  scheduled backups off-machine.

## Release Validation

Before publishing the tag, run:

```bash
ruff check .
mypy src
pytest -q
python -m safevault --help
python -m safevault --version
bash scripts/release_check.sh
```

The final Windows gate is a clean-machine install/uninstall smoke test of
`SafeVaultSetup.exe`, including onboarding, Startup, tray exit, automatic delete
capture, and restore.
